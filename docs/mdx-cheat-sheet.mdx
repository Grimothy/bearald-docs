# üìö Docusaurus MDX Cheat Sheet

Quick reference for MDX in Docusaurus.

## üìÑ File Structure & Frontmatter

Every MDX file starts with frontmatter (YAML between dashes):

```mdx
---
title: Page Title
description: Short description for SEO
sidebar_position: 1
---

# Content goes here
```

### Common Frontmatter Options

| Option | Purpose |
|--------|---------|
| `title` | The page title |
| `description` | Meta description for SEO |
| `sidebar_position` | Order in sidebar (lower first) |
| `sidebar_label` | Custom sidebar label |
| `id` | Unique identifier |

## ‚úèÔ∏è Text Formatting

### Basic Formatting

- **Bold**: `**text**` ‚Üí **text**
- *Italic*: `*text*` ‚Üí *text*
- ***Bold & Italic***: `***text***` ‚Üí ***text***
- `Inline Code`: `` `code` `` ‚Üí `code`
- ~~Strikethrough~~: `~~text~~` ‚Üí ~~text~~

### Headings

```md
# Heading 1 (avoid in content)
## Heading 2 (appears in TOC)
### Heading 3 (appears in TOC)
#### Heading 4 (document only, not TOC)
```

:::tip
Use `##` for main sections, `###` for subsections. Avoid `#` in content‚Äîit's your page title.
:::

## üñºÔ∏è Images

### Basic Image

```mdx
![Alt text for accessibility](/img/docusaurus.png)
```

Image paths are relative to the `static/` folder. So `/img/docusaurus.png` points to `static/img/docusaurus.png`.

### Centered Image

```mdx
<div style={{textAlign: 'center'}}>
  ![Alt text](/img/docusaurus.png)
</div>
```

### Centered with Width Control

```mdx
<div style={{textAlign: 'center', maxWidth: '400px', margin: '0 auto'}}>
  ![Alt text](/img/docusaurus.png)
</div>
```

### Add Padding Below Image

```mdx
<div style={{textAlign: 'center', marginBottom: '2rem'}}>
  ![Alt text](/img/logo.png)
</div>
```

:::tip
Always include descriptive alt text for accessibility. Adjust `marginBottom` value (2rem, 1.5rem, 3rem) based on desired spacing.
:::

## üìù Lists

### Unordered Lists

```md
- Item one
- Item two
  - Nested item
  - Another nested
- Item three
```

- Item one
- Item two
  - Nested item
  - Another nested
- Item three

You can also use `*` or `+` instead of `-`.

### Ordered Lists

```md
1. First item
2. Second item
   1. Nested numbered item
   2. Another nested
3. Third item
```

1. First item
2. Second item
   1. Nested numbered item
   2. Another nested
3. Third item

:::info Note
Docusaurus auto-numbers sequentially, so you can use `1.` for all items and it'll renumber correctly.
:::

## üîó Links

### Internal Links

```md
[Link to another doc](/docs/getting-started)
[Link to a page](/blog/my-post)
```

Use relative paths without file extensions.

### External Links

```md
[Docusaurus website](https://docusaurus.io)
[GitHub Repo](https://github.com/facebook/docusaurus)
```

[Docusaurus website](https://docusaurus.io)

[GitHub Repo](https://github.com/facebook/docusaurus)

### Link with Title

```md
[Docusaurus](https://docusaurus.io "Visit Docusaurus")
```

Hover text appears in the title attribute.

### Anchor Links (Jump Within Document)

```md
[Jump to Installation](#installation)

## Installation

Your content here...
```

Docusaurus auto-creates anchors from headings (lowercase, hyphens for spaces).

### Custom Anchors

```md
## My Heading {#custom-anchor}

[Link to custom](#custom-anchor)
```

Use curly braces to override the auto-generated anchor if needed.

### Quick Reference

| Type | Syntax | Example |
|------|--------|---------|
| Internal | `[text](/path)` | `[Guide](/docs/intro)` |
| External | `[text](https://url)` | `[Google](https://google.com)` |
| With Title | `[text](url "title")` | `[Link](url "Hover text")` |
| Anchor (same page) | `[text](#anchor)` | `[Jump](#installation)` |
| Custom anchor | `## Heading {#custom}` | `## Setup {#my-setup}` |

## ‚ö° Pro Tips & Gotchas

### Empty Lines Matter

MDX is sensitive to spacing. Always add blank lines before/after:

- Code blocks
- Admonitions
- Components
- Markdown lists inside components

### Closing Admonitions

```mdx
:::tip
This is a tip.
:::

Next paragraph here.
```

Every admonition **must** end with `:::` on its own line.

### Frontmatter Required

```mdx
---
title: My Page
---
```

Always include at least `title` in frontmatter. Without it, Docusaurus may have rendering issues.

### JSX and Markdown Mixing

```mdx
:::tip
Leave blank lines when mixing markdown and JSX.

<SomeComponent />

This ensures proper parsing.
:::
```

:::info
Leave blank lines when mixing markdown and JSX to ensure proper parsing.
:::
